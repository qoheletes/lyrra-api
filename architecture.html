Lyrra API Architecture

A FastAPI service for YouTube audio translation and video subtitle management. It accepts YouTube URLs, transcribes and translates speech with OpenAI Whisper, and stores structured results for fast retrieval and subtitle delivery.

FastAPI · Python 3.9+
PostgreSQL · SQLAlchemy ORM
OpenAI Whisper · Audio transcription
🏛️

Architecture

Clean / Hexagonal — domain · application · infrastructure per module

Each feature module is vertically sliced into three layers. The dependency arrow always points inward — infrastructure knows about application, application knows about domain, domain knows about nothing.

Domain
Pure Python
  • ·models.py — @dataclass entities
  • ·exceptions.py — plain Exception
  • ·sentences.py — pure algorithm (youtube)
No framework imports
Application
Orchestration
  • ·use_cases.py — classes with execute()
  • ·ports.py — typing.Protocol adapters
Depends on domain only
Infrastructure
Adapters
  • ·router.py — FastAPI + error mapping
  • ·orm_models.py — SQLAlchemy ORM
  • ·repository.py — DB queries + mapper
  • ·schemas.py — Pydantic DTOs
Depends on application + domain
Dependency Flow
infrastructure/
→
application/
→
domain/
|
app/core/
→
all layers
app/core/ (config, database, storage) is shared infrastructure — any layer may import from it. Domain is the only layer that imports from nothing outside standard Python.
Module Tree
app/ ├── core/ │ ├── config.py # Settings dataclass, loads .env │ ├── database.py # SQLAlchemy engine, Base, get_db() │ └── storage/ │ ├── client.py # StorageAdapter ABC + Local / S3 / R2 adapters │ └── exceptions.py # StorageError ├── videos/ │ ├── domain/ │ │ ├── models.py # Video, SubtitleTrackSummary dataclasses │ │ └── exceptions.py # VideoNotFoundError │ ├── application/ │ │ └── use_cases.py # GetVideoWithTracks │ └── infrastructure/ │ ├── orm_models.py # VideoORM (SQLAlchemy) │ ├── repository.py # VideoRepository → Video domain model │ ├── schemas.py # VideoOut Pydantic DTO │ └── router.py # GET /videos/{video_id} ├── subtitles/ │ ├── domain/ │ │ ├── models.py # SubtitleTrack dataclass │ │ ├── exceptions.py # TrackNotFoundError, InvalidSubtitleJSONError │ │ └── validator.py # validate_subtitle_json() — pure Python │ ├── application/ │ │ └── use_cases.py # UploadSubtitle, DeleteSubtitle, GetSubtitleTrackUrl │ └── infrastructure/ │ ├── orm_models.py # SubtitleTrackORM (SQLAlchemy) │ ├── repository.py # SubtitleRepository │ ├── schemas.py # SubtitleTrackOut Pydantic DTO │ └── router.py # POST / GET / DELETE /videos/{id}/subtitles └── youtube/ ├── domain/ │ ├── models.py # AudioDownloadResult, TranscriptionResult, WordTimestamp │ ├── exceptions.py # DownloadFailedError, TranscriptionFailedError │ └── sentences.py # Sentence, SentenceWord, build_sentences() ├── application/ │ ├── ports.py # AudioDownloader, Transcriber (typing.Protocol) │ └── use_cases.py # TranslateYouTubeVideo, GetCachedTranscription └── infrastructure/ ├── downloader.py # YtDlpAudioDownloader ├── transcriber.py # WhisperTranscriber ├── schemas.py # YouTubeTranslateRequest/Response, SentencesResponse └── router.py # POST /youtube/translate, GET /youtube/transcribe/…/sentences
ORM naming convention — SQLAlchemy classes carry an ORM suffix (VideoORM, SubtitleTrackORM) to avoid collisions with the same-named domain @dataclass entities. Repositories hold the only _to_domain() mapper that crosses between them.

Infrastructure

Core services and external dependencies

⚡

Web Framework

  • FastAPI (ASGI)
  • Uvicorn server
  • CORS for localhost:3000
  • HTTP request logging middleware
🗄️

Database

  • PostgreSQL
  • SQLAlchemy ORM
  • Alembic migrations
  • Auto-create tables on startup
🤖

AI / Transcription

  • OpenAI Whisper API
  • Model: whisper-1 (default)
  • Word-level timestamps
  • Auto-detects source language
📥

Audio Pipeline

  • yt-dlp for YouTube download
  • ffmpeg for audio extraction
  • Output: MP3 at 192kbps
  • Temp dir, cleaned after use
📦

File Storage

  • Local filesystem (default)
  • AWS S3 (configurable)
  • Served via /files static mount
  • Pluggable StorageAdapter
💾

Transcription Cache

  • JSON files on disk
  • {video_id}.json per video
  • {video_id}_sentences.json
  • Checked before calling Whisper
🗃️
